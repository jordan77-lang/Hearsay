  1. Create a new Project in ChatGPT (or a custom GPT).
  2. Save this guide as PDF and attach it under Project files / Knowledge (required).

2. What HearSay expects (import format)

HearSay Dictionary page → Template CSV or Import. The file must be UTF-8 CSV (or TSV) with a header row and these columns:

ColumnRequiredMeaning
PatternYesText as it appears in course materials — the misread token or phrase.
SpokenYesWhat NVDA should say instead (plain words, no markup).
NoteNoInstructor reminder (source slide, lab, week).
Ignore caseNoYes (default) or No.

Alternate header names accepted: Text / Substitution, Term / Pronunciation. Prefer Pattern and Spoken.

3. Accuracy: no hallucinations, ask when unsure

Wrong audio hurts students. The ChatGPT project must not invent terms or guess pronunciations for names, jargon, or ambiguous symbols.

Ground every row

  • Pattern must appear in pasted course text, an explicit instructor list, or be labeled “suggested — confirm before import.”
  • Spoken must follow instructor choice, stated department convention, or a standard you explain (e.g. “H two O” for H2O).
  • Do not bulk-fill hundreds of generic STEM rows the instructor did not ask for.

When pronunciation is unknown

  1. Do not put a guessed Spoken value in the final CSV.
  2. Offer 2–4 options in chat with brief rationale (e.g. art-history name, Italian term, acronym).
  3. Ask: “Which should students hear? Reply A, B, or C (or type your own).”
  4. Add the row to the CSV only after the instructor chooses.
  5. If they cannot decide, omit the row and tell them to add it manually in HearSay later.

Always clarify first

  • Proper nouns (artists, species, places)
  • Ambiguous abbreviations (MS, IV, BA)
  • Institutional / lab-specific coinages
  • Regional preferences (joules vs jools)

4. Project files vs instructor PDF (who uploads what)

Project owner uploads once (ChatGPT → Project → Files)

  • This guide as PDF — full rules (required)
  • hearsay-dictionary-template.csv — column format (required)
  • example-output-snippet.csv — style reference (recommended)
  • Put INSTRUCTIONS-FOR-CHATGPT-PROJECT.txt in the Instructions field only — not as a project file

Instructor uploads each course (in the chat)

  • Course PDF — syllabus, lab manual, slide export, textbook chapter
  • Optional: pasted quiz text if the PDF is scanned/image-only or equations extract badly

HearSay does not read PDFs today. ChatGPT is the PDF-scan step; HearSay is the import, edit, test, and NVDA export step.

5. PDF-first workflow (end to end)

  1. Instructor opens the project and attaches the course PDF (or pastes text).
  2. ChatGPT scans for pronunciation risks: abbreviations, units, formulas, symbols, subscripts, proper nouns, field jargon — not every English word.
  3. ChatGPT shows a draft term list in chat (with page refs). Uncertain terms get 2–4 pronunciation options; instructor picks before CSV.
  4. ChatGPT delivers confirmed CSV only.
  5. Instructor saves CSV (e.g. chem113-pronunciations.csv).
  6. HearSay → Dictionary → ☁ Connect → Import → ▶ review rows → Save class.
  7. Refine in HearSay over the term; use Screen Reader Lab to test plain quiz text.
  8. Download for students → NVDA add-on + install PDF (NVDA 2026.1+).
  9. Students install once; test in Notepad: kJ/mol, 10 mL, J/g°C.

6. How to write Pattern and Spoken

6.1 Pattern = atomic tokens (not sentences)

Default rule. One CSV row = one token screen readers misread (acronym, unit, symbol, variable, formula). Not whole equations, not lab-specific quantities like 50 g, not rows where Spoken equals Pattern.
  • Include: NASA, mL, J/g°C, ΔT, qcalorimeter, H₂O₂, monopropellant (when Spoken changes).
  • Exclude: qcalorimeter = Ccalorimeter × ΔT, ΔT = T2 − T1, 200 g, 100 kJ — atomic rows + HearSay composition (or NVDA regex for number+unit) cover these.
  • Canvas equations: point instructors to MathSay (/mathsay/) for MathML and dual-notation export — not dictionary equation rows.
  • Copy Patterns from real stems, not from memory. Symbols exactly: J/g°C, not “joules per gram C”.
  • Separate rows for variants: DI water, DIwater; T2, T₂; qcalorimeter, q_solution if both appear.

6.2 Spoken = “what the ear should hear”

  • Lowercase prose unless capitalization matters.
  • Separate words with spaces: jools per gram degree Celsius.
  • Chemical formulas: letter + count: H two O, C U S O 4.
  • Variables: T of 2, q of solution, capital C of calorimeter.
  • Greek: delta T, delta H.
  • Avoid IPA symbols in Spoken unless testing with a specific voice; use phonetic English.

6.3 Ignore case

Use Yes for units, formulas, and symbols. Use No when case distinguishes terms (e.g. capital M for molarity in “3 M HCl” → spoken moh lurr; capital Ccalorimeter for heat capacity vs lowercase c for specific heat).

How instructors should format documents

  • Digit subscripts: unicode T₂, T₁ in Word/Canvas when possible; add CSV rows for plain T2, T1 if the PDF uses them.
  • Word subscripts: glued plain text — msolution, qcalorimeter, csolution — not m solution with a space. Spreadsheet refs B4–B7 stay plain (not subscripts).
  • Formula subscripts: cH₂O, cH₂ with unicode H₂; add cH2O if the export is ASCII-only.
  • Invisible characters: Google Docs paste may include zero-width spaces — strip from Pattern or warn the instructor.

Spoken naming (ChatGPT must follow)

  • q… variables → q of … (e.g. q of solution)
  • m… mass → m sub … (e.g. m sub solution, m sub peroxide solution)
  • c… specific heat word subscript → c sub …; formula subscript → c of H two O
  • Capital C… heat capacity → capital C of calorimeter (Ignore case: No)
  • ΔT → delta T

Variant checklist per lab (tokens only)

Add a row for each token form that appears: T2 and T₂; qcalorimeter and q_calorimeter; J/g°C and J/°C; H2O2 and H₂O₂. Do not add whole-equation rows (e.g. ΔT = T2 − T1) unless the instructor confirms Screen Reader Lab still misreads after atomic rows are imported.

Chemistry classes in HearSay can merge bundled regex rules for units after numbers (e.g. 10 mL). Instructor CSV should list unit tokens (mL, g, kJ) — not every number+unit pair from the lab (e.g. not 50 g, 200 g).

Biology

  • Latin species names: slow phonetic English (Homo sapiens → HO mo SAP ee enz — confirm with instructor).
  • Gene / allele notation: avoid breaking English words; only pattern real tokens from materials.
  • Units: μL, g, mL same as chemistry.
  • Pathway acronyms: spell (NADH → N A D H).

Mathematics

  • Dictionary: atomic symbols and variables only (Δx, x^2 if flat text).
  • Canvas: stacked fractions and equations → MathSay (/mathsay/) for MathML or dual-notation HTML — not whole-equation CSV rows.
  • Plain text: x^2 → x squared; Δx → delta x.
  • Parentheses: spoken open parenthesis / close parenthesis when symbols are skipped.
  • Never add entire equations to the CSV by default — even if they appear verbatim in the PDF.

Art history

  • Artist names and titles: offer 2–4 spoken options; add to CSV only after instructor picks.
  • Movements and periods: confirm pronunciation (e.g. Baroque) — do not guess.

Music theory

  • Italian terms and dynamics: offer options or ask instructor preference.
  • Chords and Roman numerals: context matters — clarify before CSV.

9. Quality checklist (ChatGPT before delivery)

  • Header row exactly: Pattern, Spoken, Note, Ignore case
  • No blank Pattern or Spoken cells
  • Every row is an atomic token — not a sentence, equation, or lab-specific quantity (50 g)
  • No row where Spoken equals Pattern (omit no-op rows)
  • No whole-equation rows unless instructor confirmed Lab failure after atomic import
  • No truncated or incomplete Patterns from PDF extract
  • Every Pattern grounded in instructor text or explicit approval
  • Uncertain terms confirmed via choices — not guessed in CSV
  • Spoken is natural TTS English, not abbreviations unless intentional
  • No regex in Pattern unless instructor is advanced
  • Includes 5–10 Notepad test tokens (not whole paragraphs)
  • Notes cite source when possible
  • Short summary: token count, equation rows omitted, pending items

10. What ChatGPT should not do

  • Output HearSay HTML, Canvas markup, or NVDA .dic files unless explicitly asked.
  • Promise fixes inside Google Docs equation objects.
  • Replace institution-wide accessibility process — dictionary supplements good authoring.
  • Use IPA in Spoken column without instructor approval.
  • Guess pronunciations for proper nouns or ambiguous symbols
  • Present uncertain terms as final CSV rows without instructor choice
  • Invent terms that never appeared in course materials
  • Add whole equations, lab quantity rows (200 g), or no-op rows (Spoken = Pattern) by default
  • Add truncated PDF equation extracts

11. After import on HearSay

  1. Use ▶ on rows to hear preview (Web Speech approximates NVDA; real test is NVDA).
  2. Use Screen Reader Lab to paste plain quiz text and compare with/without dictionary.
  3. Export NVDA add-on; bump version each term.
  4. Post add-on + student install PDF on Canvas.
